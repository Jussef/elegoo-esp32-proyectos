Descripción
Avanzado Sensores · Servos · I2C

Un gimbal es una plataforma que se mantiene nivelada aunque inclines la base. El GY-6500 (MPU-6050) mide el ángulo de Roll y un único servo SG90 gira en sentido contrario para compensar el movimiento, manteniendo estable un eje de una cámara o cualquier objeto. Es la base de los estabilizadores que usan drones y cámaras de mano.

Alimentación del servo: Un SG90 moviéndose puede pedir picos de corriente que reinician el ESP-32 si lo alimentas desde su pin 5V. Usa la fuente de 5V externa para el servo y une todos los GND (ESP-32 + fuente) en un punto común.
Código Arduino
C++ / Arduino IDE (ESP-32) 
#include <Wire.h>
#include <ESP32Servo.h>
#include <math.h>

#define MPU_ADDR    0x68
#define PIN_SERVO   13

Servo servoRoll;

void mpuInit() {
  Wire.beginTransmission(MPU_ADDR);
  Wire.write(0x6B);  // Registro PWR_MGMT_1
  Wire.write(0x00);  // Despertar el sensor (sale del modo sleep)
  Wire.endTransmission(true);
}

void readAccel(float &ax, float &ay, float &az) {
  Wire.beginTransmission(MPU_ADDR);
  Wire.write(0x3B);              // ACCEL_XOUT_H
  Wire.endTransmission(false);
  Wire.requestFrom(MPU_ADDR, 6, true);

  uint8_t d[6];
  for (int i = 0; i < 6; i++) d[i] = Wire.read();

  ax = (int16_t)(d[0] << 8 | d[1]) / 16384.0f;  // Rango +-2g (16384 LSB/g)
  ay = (int16_t)(d[2] << 8 | d[3]) / 16384.0f;
  az = (int16_t)(d[4] << 8 | d[5]) / 16384.0f;
}

void setup() {
  Serial.begin(115200);
  Wire.begin();   // SDA=GPIO21, SCL=GPIO22
  mpuInit();

  servoRoll.attach(PIN_SERVO);

  // Posicion central de reposo
  servoRoll.write(90);
  delay(500);
  Serial.println("Gimbal listo");
}

void loop() {
  float ax, ay, az;
  readAccel(ax, ay, az);

  // Angulo de Roll respecto a la gravedad (inclinacion lateral)
  float roll = atan2f(ay, sqrtf(ax*ax + az*az)) * 180.0f / M_PI;

  // Filtro paso-bajo para suavizar (evita vibracion en el servo)
  static float rf = 0.0f;
  rf = rf * 0.85f + roll * 0.15f;

  // El servo compensa en sentido CONTRARIO a la inclinacion.
  // 90 = centro. Se limita a +-45 grados para no forzar el servo.
  int posRoll = 90 - (int)constrain(rf, -45.0f, 45.0f);

  servoRoll.write(posRoll);

  Serial.printf("Roll:%+.1f  ->  Servo:%d\n", rf, posRoll);

  delay(20);   // ~50 actualizaciones por segundo
}
Cómo usar
1
Instala la librería ESP32Servo desde el Gestor de Librerías. El GY-6500 se lee directamente por I2C con Wire.h, no necesita librería extra.
2
Monta el sensor GY-6500 sobre la plataforma móvil (la parte que quieres mantener nivelada), y el servo sobre la base fija.
3
Alimenta el servo con la fuente de 5V externa y une todos los GND. Sube el código.
4
Inclina la base de lado a lado: el servo gira en sentido contrario para que la plataforma se mantenga horizontal en ese eje.
5
Abre el Monitor Serie a 115200 baudios para ver el ángulo medido y la posición del servo en tiempo real.
Tip: Si el servo compensa al revés (la plataforma se inclina más en vez de corregir), cambia 90 - por 90 + en esa línea. Si el servo vibra demasiado, sube el factor del filtro de 0.85f a 0.90f para suavizar más. ¿Quieres estabilizar el otro eje? Usa pitch = atan2f(ax, sqrtf(ay*ay + az*az)) en lugar del cálculo de roll.
