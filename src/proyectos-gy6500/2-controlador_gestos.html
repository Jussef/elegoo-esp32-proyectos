Descripción
Intermedio Sensores · LED RGB · I2C

Convierte la orientación del módulo en color. El GY-6500 (MPU-6050) mide cómo lo inclinas y el código mapea los ángulos de Pitch y Roll a los canales Rojo, Verde y Azul de un LED RGB. Inclina hacia adelante, atrás o a los lados y el color cambia suavemente, como un cubo de colores que respondes con la mano.

Código Arduino
C++ / Arduino IDE (ESP-32) 
#include <Wire.h>
#include <math.h>

#define MPU_ADDR  0x68
#define PIN_R     25
#define PIN_G     26
#define PIN_B     27

// Canales PWM del ESP-32 (LEDC)
#define CH_R  0
#define CH_G  1
#define CH_B  2

void mpuInit() {
  Wire.beginTransmission(MPU_ADDR);
  Wire.write(0x6B);  // Registro PWR_MGMT_1
  Wire.write(0x00);  // Despertar el sensor (sale del modo sleep)
  Wire.endTransmission(true);
}

void readAccel(float &ax, float &ay, float &az) {
  Wire.beginTransmission(MPU_ADDR);
  Wire.write(0x3B);              // ACCEL_XOUT_H
  Wire.endTransmission(false);
  Wire.requestFrom(MPU_ADDR, 6, true);

  uint8_t d[6];
  for (int i = 0; i < 6; i++) d[i] = Wire.read();

  ax = (int16_t)(d[0] << 8 | d[1]) / 16384.0f;  // Rango +-2g (16384 LSB/g)
  ay = (int16_t)(d[2] << 8 | d[3]) / 16384.0f;
  az = (int16_t)(d[4] << 8 | d[5]) / 16384.0f;
}

void setup() {
  Serial.begin(115200);
  Wire.begin();   // SDA=GPIO21, SCL=GPIO22
  mpuInit();

  // Configura 3 canales PWM a 5 kHz, resolucion de 8 bits (0-255)
  ledcSetup(CH_R, 5000, 8);  ledcAttachPin(PIN_R, CH_R);
  ledcSetup(CH_G, 5000, 8);  ledcAttachPin(PIN_G, CH_G);
  ledcSetup(CH_B, 5000, 8);  ledcAttachPin(PIN_B, CH_B);

  Serial.println("Controlador de gestos listo. Inclina el modulo.");
}

void loop() {
  float ax, ay, az;
  readAccel(ax, ay, az);

  // Angulos de inclinacion -90..+90 grados
  float pitch = atan2f(ax, sqrtf(ay*ay + az*az)) * 180.0f / M_PI;
  float roll  = atan2f(ay, sqrtf(ax*ax + az*az)) * 180.0f / M_PI;

  // Filtro paso-bajo para que el color cambie suave
  static float pf = 0.0f, rf = 0.0f;
  pf = pf * 0.8f + pitch * 0.2f;
  rf = rf * 0.8f + roll  * 0.2f;

  // Mapea los gestos a color:
  //  - ROJO  segun el Pitch (inclinar adelante/atras)
  //  - AZUL  segun el Roll  (inclinar izquierda/derecha)
  //  - VERDE mas brillante cuanto mas NIVELADO este
  int r = map((long)constrain(fabsf(pf), 0.0f, 90.0f), 0, 90, 0, 255);
  int b = map((long)constrain(fabsf(rf), 0.0f, 90.0f), 0, 90, 0, 255);
  int g = map((long)constrain(fabsf(pf) + fabsf(rf), 0.0f, 90.0f), 0, 90, 255, 0);

  ledcWrite(CH_R, r);
  ledcWrite(CH_G, g);
  ledcWrite(CH_B, b);

  Serial.printf("Pitch:%+.0f Roll:%+.0f  ->  R:%3d G:%3d B:%3d\n",
                pf, rf, r, g, b);

  delay(30);
}
Cómo usar
1
No necesitas ninguna librería extra: el LED se controla con el PWM nativo del ESP-32 (ledc) y el GY-6500 por I2C con Wire.h.
2
Conecta cada pata de color del LED RGB a su GPIO con una resistencia de 220Ω, y la pata común (cátodo) a GND.
3
Sube el código. Con el módulo plano y quieto el LED se verá verde (nivelado).
4
Inclina adelante/atrás para añadir rojo, e izquierda/derecha para añadir azul. Mezcla ambos gestos para obtener morados y blancos.
5
Abre el Monitor Serie a 115200 baudios para ver los ángulos y los valores R/G/B que se están enviando al LED.
Tip: Cambia el mapeo de colores a tu gusto. Por ejemplo, puedes asignar el ángulo completo (con signo) al matiz de un solo color, o usar un tercer eje para el brillo. Si el LED queda muy tenue, baja las resistencias a 150Ω.
