Descripción
Demo de prueba IR Emitter + Receiver

Tienes dos módulos separados. Usa las pestañas de abajo para probar cada uno: el Emisor es el que parece un LED (pines S / medio / −) y lo verificarás con la cámara del celular; el Receptor es el del vidrio negro (pines G / R / Y) y lo probarás leyendo los botones de tu control remoto.

Ambas pruebas usan la librería "IRremote" de Armin Joachimsmeyer (v4.x). Instálala una vez desde el Gestor de Librerías; por eso el include es IRremote.hpp (con .hpp).
Conexiones · Emisor (el que parece un LED)
Pin del móduloPin ESP-32Notas
S (señal)GPIO 23Pin de datos
Medio— sin conectarEste módulo no usa el pin del centro
− (GND)GND—
Solo necesita 2 cables: S y −. El pin del medio se deja libre.
Código · Emisor
C++ / Arduino IDE (ESP-32) 
#include <IRremote.hpp>   // libreria "IRremote" v4.x

#define PIN_SEND 23        // pin de senal (S) del EMISOR

void setup() {
  Serial.begin(115200);
  IrSender.begin(PIN_SEND);
  Serial.println("Emisor IR listo: enviando un codigo cada segundo.");
}

void loop() {
  // Envia un codigo NEC conocido (direccion 0x00, comando 0x45)
  IrSender.sendNEC(0x00, 0x45, 0);
  Serial.println("Enviado 0x45");
  delay(1000);
}
Cómo probar el emisor
1
Conecta S→GPIO 23 y −→GND. El pin del medio queda libre.
2
Sube el código y abre el Monitor Serie a 115200: verás Enviado 0x45 repetido.
3
La luz IR es invisible al ojo. Apunta la cámara del celular al LED del emisor: verás un destello morado parpadeando cada segundo. ¡Eso confirma que emite!
Prueba real (opcional): apunta el emisor al receptor de la otra pestaña (a 2-5 cm) y corre allí el código de recepción: deberías ver llegar el 0x45.
Conexiones · Receptor (el del vidrio negro)
Pin del móduloPin ESP-32Notas
Y (señal / Yellow)GPIO 4Pin de datos
R (VCC / Red)3.3VAlimentación (funciona a 3.3V o 5V)
G (GND)GND—
Ojo con el mapeo G/R/Y: la convención normal es G=GND, R=VCC(+) y Y=señal. Si tu placa no responde, no se daña: prueba a intercambiar y verifica las letras impresas junto a cada pin.
Código · Receptor (lee el control remoto)
C++ / Arduino IDE (ESP-32) 
#include <IRremote.hpp>   // libreria "IRremote" v4.x

#define PIN_RECV 4         // pin de senal (Y) del RECEPTOR

void setup() {
  Serial.begin(115200);
  IrReceiver.begin(PIN_RECV, ENABLE_LED_FEEDBACK);
  Serial.println("Receptor listo. Apunta el control y presiona botones.");
}

void loop() {
  // Cuando llega una senal IR completa, la decodifica
  if (IrReceiver.decode()) {
    // Codigo del boton presionado (en hexadecimal)
    Serial.print("Boton -> 0x");
    Serial.println(IrReceiver.decodedIRData.command, HEX);

    // Resumen completo (protocolo, direccion, datos crudos)
    IrReceiver.printIRResultShort(&Serial);

    IrReceiver.resume();   // listo para el siguiente boton
  }
}
Cómo probar el receptor
1
Conecta Y→GPIO 4, R→3.3V y G→GND.
2
Quítale la lengüeta de plástico a la pila del control remoto si es nuevo.
3
Sube el código y abre el Monitor Serie a 115200 baudios.
4
Apunta el control al receptor y presiona botones: verás el código de cada tecla, p. ej. Boton -> 0x45. Anótalos, los necesitarás en tus proyectos.
¿Sale 0x0 o "Unknown"? Suele pasar al mantener un botón (envía códigos de repetición) o al apuntar de lado. Da toques cortos y de frente al sensor.
