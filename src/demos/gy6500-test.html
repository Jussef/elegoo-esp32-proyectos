Descripción
Demo de prueba Sensor · I2C

La prueba más sencilla para tu módulo GY-6500 (MPU-6050): lee el acelerómetro (3 ejes), el giroscopio (3 ejes) y la temperatura interna, y lo imprime todo en el Monitor Serie. Calcula además los ángulos de inclinación (Pitch y Roll). Sirve para confirmar que el sensor está bien conectado por I2C, que responde en su dirección (0x68) y que los valores cambian al mover el módulo. No necesita pantalla ni librerías externas.

Conexiones
ComponentePin ESP-32Notas
GY-6500 — VCC3.3VEl módulo lleva regulador, pero 3.3V es lo más seguro
GY-6500 — GNDGND—
GY-6500 — SDAGPIO 21Bus I2C (datos)
GY-6500 — SCLGPIO 22Bus I2C (reloj)
Solo 4 cables. Los pines AD0, INT, XDA, XCL del módulo se dejan sin conectar para esta prueba. Con AD0 a GND (lo normal) la dirección I2C es 0x68; si lo conectaras a 3.3V sería 0x69.
Código Arduino
C++ / Arduino IDE (ESP-32) 
#include <Wire.h>
#include <math.h>

#define MPU_ADDR  0x68   // Direccion I2C del MPU-6050 (AD0 a GND)

void mpuWrite(uint8_t reg, uint8_t val) {
  Wire.beginTransmission(MPU_ADDR);
  Wire.write(reg);
  Wire.write(val);
  Wire.endTransmission(true);
}

void setup() {
  Serial.begin(115200);
  delay(300);
  Wire.begin();          // SDA=GPIO21, SCL=GPIO22 en el ESP-32

  // Comprobar que el sensor responde en el bus I2C
  Wire.beginTransmission(MPU_ADDR);
  if (Wire.endTransmission() != 0) {
    Serial.println("ERROR: GY-6500 no responde. Revisa cables y la direccion 0x68/0x69.");
    while (true) delay(1000);
  }

  mpuWrite(0x6B, 0x00);  // PWR_MGMT_1: despertar el sensor (sale del modo sleep)
  Serial.println("GY-6500 (MPU-6050) detectado. Iniciando lecturas...\n");
  delay(500);
}

void loop() {
  uint8_t d[14];

  // Leer de golpe 14 registros: Accel(6) + Temp(2) + Gyro(6)
  Wire.beginTransmission(MPU_ADDR);
  Wire.write(0x3B);                 // ACCEL_XOUT_H
  Wire.endTransmission(false);
  Wire.requestFrom(MPU_ADDR, 14, true);
  for (int i = 0; i < 14; i++) d[i] = Wire.read();

  // Acelerometro en g (rango +-2g -> 16384 LSB/g)
  float ax = (int16_t)(d[0]  << 8 | d[1])  / 16384.0f;
  float ay = (int16_t)(d[2]  << 8 | d[3])  / 16384.0f;
  float az = (int16_t)(d[4]  << 8 | d[5])  / 16384.0f;

  // Temperatura en grados C (formula de la hoja de datos)
  float tempC = (int16_t)(d[6] << 8 | d[7]) / 340.0f + 36.53f;

  // Giroscopio en grados/segundo (rango +-250 -> 131 LSB/(deg/s))
  float gx = (int16_t)(d[8]  << 8 | d[9])  / 131.0f;
  float gy = (int16_t)(d[10] << 8 | d[11]) / 131.0f;
  float gz = (int16_t)(d[12] << 8 | d[13]) / 131.0f;

  // Angulos de inclinacion a partir de la gravedad
  float pitch = atan2f(ax, sqrtf(ay*ay + az*az)) * 180.0f / M_PI;
  float roll  = atan2f(ay, sqrtf(ax*ax + az*az)) * 180.0f / M_PI;

  Serial.printf("Accel[g]  X:%+.2f  Y:%+.2f  Z:%+.2f\n", ax, ay, az);
  Serial.printf("Gyro[d/s] X:%+.1f  Y:%+.1f  Z:%+.1f\n", gx, gy, gz);
  Serial.printf("Pitch:%+.1f  Roll:%+.1f   Temp:%.1f C\n", pitch, roll, tempC);
  Serial.println("----------------------------------------");

  delay(500);
}

En reposo y plano verás Z ≈ +1.00 g (la gravedad) y los demás cerca de 0. Al inclinar el módulo, Pitch y Roll cambian; al girarlo rápido, el Gyro se dispara y vuelve a 0 al detenerlo. Si todo eso pasa, ¡tu sensor funciona!
Cómo usar
1
Conecta el GY-6500: VCC→3.3V, GND→GND, SDA→21 y SCL→22. Solo 4 cables.
2
No hace falta instalar ninguna librería: el código lee el sensor directamente por I2C con Wire.h (ya viene con el core del ESP-32).
3
Sube el código y abre el Monitor Serie a 115200 baudios.
4
Mueve e inclina el módulo con la mano: los valores de Accel, Gyro, Pitch y Roll deben reaccionar al instante.
¿Sale "GY-6500 no responde"? Revisa que SDA y SCL no estén intercambiados, que la alimentación sea 3.3V y que el módulo esté bien sentado en el protoboard. Si tu placa tiene AD0 conectado a 3.3V, cambia 0x68 por 0x69.
